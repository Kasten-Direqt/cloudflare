HTTP Message Signatures Directory Documentation

Overview

This API implements the HTTP Message Signatures Directory format as specified in the IANA media types registry. The content type application/http-message-signatures-directory+json is used for serving a directory of available HTTP message signatures.

Endpoint

GET /signatures-directory

Returns a directory of HTTP message signatures available for validation, including the cryptographic keys used for signing.

Response Format

The response is a JSON object with the following structure:

{
  "signatures": [
    {
      "id": "sig1",
      "algorithm": "ed25519",
      "key_id": "[key identifier]",
      "created": "2025-07-10T19:30:00Z",
      "expires": "2025-07-11T19:30:00Z",
      "headers": ["(created)", "(expires)", "host", "date", "content-type"]
    }
  ],
  "keys": [
    {
      "kid": "[key identifier]",
      "kty": "OKP",
      "crv": "Ed25519",
      "x": "[public key]",
      "nbf": 1752174878083
    }
  ]
}

Fields Description

Signatures Object

Field Description
id The identifier for the signature
algorithm The algorithm used for signing (e.g., ed25519, rsa-v1_5-sha256)
key_id Reference to the key used for signing (matches a kid in the keys array)
created ISO 8601 timestamp when the signature was created
expires ISO 8601 timestamp when the signature expires
headers Array of HTTP header fields included in the signature

Keys Object

Field Description
kid Key identifier (JWK Thumbprint)
kty Key type (e.g., "OKP" for Octet Key Pair)
crv Curve used for the key (e.g., "Ed25519")
x Public key data (base64url encoded)
nbf Not Before timestamp (milliseconds since epoch)

IANA Registration

The media type application/http-message-signatures-directory+json is registered with IANA in the "Media Types" registry.
